# 동네꿀팁 고도화 설계 및 구현 플랜 (Phase 2)

이 문서는 동네꿀팁(`dongnae.html`)의 기능 확장을 위한 단계별 작업 계획을 담고 있습니다. 사용자 편의성 증대, 익명 제보 권한 관리, 풍부한 시각 자료 제공에 중점을 둡니다.

## 🎯 1. 핵심 목표 및 요구 사항

1. **익명 제보 관리 권한**: 로그인 없이 제보한 글도 본인이 직접 수정 및 삭제할 수 있어야 합니다.
2. **네이버 연관 사진 표출 및 쿼터 최적화**: 사용자가 직접 사진을 업로드하는 기능은 제외하고, 네이버 API를 활용해 상호명과 연관된 이미지를 자동으로 불러옵니다. 초기 5장을 먼저 보여주고 더보기로 전체를 확인할 수 있도록 하며, 일일 한도(25,000회) 방어를 위해 강력한 캐싱 로직을 구현합니다.
3. **네이버 지도 앱/웹 아웃링크**: '주변 인기 꿀팁 TOP 10' 리스트 및 마커 클릭 시 나타나는 팝업에서 "네이버 지도에서 보기" 버튼을 제공하여, 실제 길찾기나 리뷰 확인 등 앱 전환을 유도합니다.

## 🛠 2. 세부 구현 방안

### 2.1 본인 글 인증(수정/삭제 권한) 로직
완벽한 익명성을 보장하면서도 본인 인증을 처리하기 위해 **Device Fingerprint(디바이스 지문) 및 난수 토큰 방식**을 결합하여 사용합니다.

- **등록 시점:** 제보 글을 등록할 때, 클라이언트가 고유 난수(UUID 또는 `Math.random` 해시) 형태의 `authorToken`을 생성합니다.
- **저장 구조:**
  - DB(Firestore)에는 문서 데이터와 함께 `authorToken` 필드를 저장합니다.
  - 사용자의 로컬 환경(`localStorage`)에는 `{ "tipId_123": "authorToken_abc" }` 형태로 권한 키셋을 기록해 둡니다.
- **수정/삭제 시점:** 수정/삭제 버튼을 누를 때 클라이언트의 `localStorage`에서 해당 글의 `authorToken`이 일치하는지 판별하여 UI 버튼 노출 여부 및 DB 업데이트 접근 권한을 판단합니다.

### 2.2 네이버 API 연관 사진 표출 및 Edge 캐싱 (비용 0원 최적화)
Firestore에 이미지 URL을 저장하면 문서 쓰기(Write) 및 스토리지 용량 증가에 따른 미세한 과금이 누적될 수 있습니다. 이를 방지하고 네이버 API 일일 한도(25,000회)를 방어하기 위해 **Cloudflare Worker의 Edge Cache**를 적극 활용합니다.

- **표출 UI (InfoWindow 내 갤러리):**
  - 마커 정보창 하단에 네이버 이미지 검색 결과를 가로 스크롤 또는 썸네일 그리드 형태로 표출합니다.
  - **초기 노출:** 최대 5장의 이미지만 화면에 먼저 렌더링하여 로딩 속도와 트래픽을 최적화합니다.
  - **더보기 버튼:** 5장 이상의 결과가 있을 경우 `더보기` 버튼을 노출하고, 클릭 시 나머지 전체 이미지를 펼쳐볼 수 있게 합니다.

- **Cloudflare Edge Cache 100% 활용 (캐시 히트/미스):**
  1. 클라이언트는 마커 클릭 시 단순히 워커 프록시로 GET 요청만 보냅니다. 예: `https://aitomo-navermap.../image?query={상호명}`
  2. **워커 내부 캐싱 로직:** 워커가 요청을 받으면 `caches.default.match()`를 통해 Cloudflare 엣지 노드에 이미 동일한 검색 결과(JSON)가 있는지 확인합니다.
  3. **Cache Miss:** 엣지에 데이터가 없다면 네이버 API를 1회 호출하여 결과를 받아오고, 이 응답에 `Cache-Control: public, max-age=2592000` (30일) 헤더를 붙여 엣지 캐시에 저장 후 클라이언트에게 반환합니다.
  4. **Cache Hit:** 엣지에 데이터가 있다면 네이버 API를 아예 호출하지 않고 엣지 노드에서 캐시된 JSON 결과를 빛의 속도로 0원에 반환합니다.
  - *결과적으로 Firestore Write 비용 0원, 네이버 API 호출 횟수 극단적 절감 효과를 달성합니다.*

### 2.3 "네이버 지도로 바로 보기" 아웃링크 버튼
외부 지도 서비스로 사용자를 즉시 연결합니다.

- **적용 위치:** 
  1. 주변 인기 꿀팁 TOP 10 리스트 아이템 내부 (작은 아이콘 형태)
  2. 지도 마커 클릭 시 나오는 InfoWindow 하단 ("공감" 버튼 옆 분할 뷰 형태)
- **URL 규칙:** 
  - 특정 좌표 검색: `https://map.naver.com/p/search/{상호명}` URI 인코딩.
  - 브라우저나 앱이 설치된 모바일 기기라면 이 링크를 눌렀을 때 자동으로 네이버 지도 앱으로 전환되거나 웹 페이지로 매끄럽게 연결됩니다.
