# AiTomo - 동네꿀팁 배포 및 보안 설정 가이드

본 문서는 동네꿀팁(Phase 5) 기능의 성공적인 런칭을 위한 **보안 규칙(Security Rules)** 및 **배포 설정(Cloudflare/Firebase)** 에 대한 가이드입니다.

---

## 1. Firebase Firestore 보안 규칙 (Security Rules)

클라이언트(브라우저)에서 직접 Firestore DB에 접근하므로, 악의적인 데이터 변조나 무단 삭제를 막기 위해 보안 규칙 설정이 필수입니다.
Firebase Console (`빌드` > `Firestore Database` > `규칙` 탭)에 접속하여 아래의 코드로 덮어쓰기 해주세요.

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // [공통] 익명 기반 서비스이므로 읽기/쓰기는 허용하되, 데이터 검증을 수행
    
    // 1. 커피라운지 (익명 게시판)
    match /coffeeLounge/{postId} {
      allow read: if true;
      allow create: if request.resource.data.keys().hasAll(['title', 'createdAt']);
      allow update: if request.resource.data.diff(resource.data).affectedKeys()
                      .hasOnly(['likes', 'comments', 'content']);
      allow delete: if false; // 클라이언트 직접 삭제 방지 (관리자 콘솔에서만)
      
      // 커피라운지 댓글 (Sub-collection)
      match /replies/{replyId} {
        allow read: if true;
        allow create: if request.resource.data.keys().hasAll(['content', 'createdAt']);
        allow update, delete: if false; 
      }
    }

    // 2. 동네꿀팁 (새로 추가된 콜렉션)
    match /dongnaeTips/{tipId} {
      // 누구나 꿀팁 조회 가능
      allow read: if true;
      
      // 생성: 필수 필드(상호명, 카테고리, 좌표 등)가 존재해야만 허용
      allow create: if request.resource.data.keys().hasAll(['placeName', 'category', 'lat', 'lng', 'status', 'createdAt'])
                    && request.resource.data.status == 'active';
      
      // 업데이트: 좋아요(likes), 좋아요 누른 사람(likedBy) 배열만 수정 가능하도록 제한 (상호명 변경 등 방지)
      allow update: if request.resource.data.diff(resource.data).affectedKeys().hasOnly(['likes', 'likedBy']);
      
      // 삭제 방지
      allow delete: if false;
    }
    
  }
}
```

## 2. Cloudflare 배포 및 Worker 환경변수 설정

프론트엔드에서 네이버 API를 직접 호출하면 CORS 에러 및 API 키 유출 문제가 발생합니다. 이를 위해 만들어둔 Proxy 스크립트(`workers/naver-local-proxy.js`)를 배포해야 합니다.

### 2-1. Worker 배포하기
터미널(명령 프롬프트)을 열고 프로젝트 루트 폴더에서 아래 명령어를 순서대로 실행하세요.

1. **로그인 및 배포 준비**
   ```bash
   npx wrangler login
   ```
2. **Worker 배포 (최초 1회)**
   ```bash
   npx wrangler deploy workers/naver-local-proxy.js --name aitomo-navermap --compatibility-date 2026-04-22
   ```
   *배포가 완료되면 `https://aitomo-navermap.cdy3088.workers.dev` 형태의 주소가 발급됩니다.*

### 2-2. 환경변수(Secret Key) 주입하기
배포된 Worker가 네이버 서버와 통신할 수 있도록 API 키를 안전하게 주입합니다. (이 키들은 코드에 남지 않고 Cloudflare 서버에 암호화되어 저장됩니다.)

```bash
# 1. Client ID 주입 (프롬프트가 뜨면 oc3qe63frx 입력 후 엔터)
npx wrangler secret put NAVER_CLIENT_ID --name aitomo-navermap

# 2. Client Secret 주입 (프롬프트가 뜨면 bQsnPVbZc9Vn4ekAdobBNpi7QG7GRzUJKoOkRolx 입력 후 엔터)
npx wrangler secret put NAVER_CLIENT_SECRET --name aitomo-navermap
```

### 2-3. 클라이언트 URL 수정
만약 Worker 주소가 `aitomo-navermap.cdy3088.workers.dev`와 달라졌다면, `js/dongnae.js` 의 fetch URL을 배포된 본인의 Worker 주소로 변경해 주시면 완벽하게 연동됩니다.

## 3. 로컬 환경 테스트 (`run_server.bat`)

배포 전 로컬에서 모든 기능이 정상 작동하는지 확인합니다.

1. `run_server.bat` 파일을 더블클릭하여 로컬 서버를 실행합니다.
2. `http://127.0.0.1:8080/dongnae.html` 에 접속합니다.
3. **[체크리스트]**
   - [ ] 지도 로딩 시 현재 내 위치로 파란색 점이 찍히는지 확인
   - [ ] [제보하기] 버튼을 눌러 모달이 자연스럽게 올라오는지 확인
   - [ ] 상호명 검색 및 카테고리 지정 후 제보 등록이 되는지 확인 (현재는 로컬 Fallback 더미 데이터로 검색이 진행됨)
   - [ ] 제보 직후 지도 위에 지정된 마커가 생기는지 확인
   - [ ] 마커 클릭 시 인포윈도우(팝업)가 뜨고, 공감(❤️) 버튼을 눌렀을 때 숫자가 올라가는지 확인

위 테스트가 모두 통과되었다면, 정적 파일(HTML, JS, CSS)들을 기존처럼 Cloudflare Pages나 호스팅 서버에 업로드하여 배포를 마치시면 됩니다! 🎉
