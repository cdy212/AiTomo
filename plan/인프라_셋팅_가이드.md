# 실 서비스 가동을 위한 인프라 셋팅 가이드 (Firebase & Cloudflare)

현재 프로젝트는 `dataService.js`를 통해 내부 로직 추상화를 끝마친 상태입니다. 임시 로컬 캐시 모드가 아닌 실제 클라우드 인프라를 연결하기 위해서 이 가이드 라인을 따라 **Firebase(데이터베이스)** 및 **Cloudflare(웹 호스팅)** 세팅을 진행해 주세요.

---

## 1. Firebase 설정 가이드 (실 데이터 저장 및 읽기)

Firebase 콘솔에서 새 프로젝트를 생성하고, 보안이 적용된 데이터베이스를 가동하는 과정입니다.

### 1-1. Firebase 프로젝트 생성
1. [Firebase Console](https://console.firebase.google.com/) 에 접속하여 **[프로젝트 추가(Add Project)]** 버튼을 클릭합니다.
2. 프로젝트 이름을 입력합니다. (예: `aitomo-app`)
3. `Google Analytics` 사용 여부를 결정합니다. (안 하셔도 무방합니다)
4. 약관 동의 후 **프로젝트 만들기**를 클릭합니다.

### 1-2. 웹 앱(Web App) 생성 및 Config 값 추출
1. 생성된 프로젝트 메인 화면에서 로고들 중 **`</>` (웹 마크)** 아이콘을 클릭합니다.
2. 앱 닉네임을 설정하고 (예: `aitomo-web`) **앱 등록**을 클릭합니다.
3. 화면에 나타나는 소스코드 중 `const firebaseConfig = { ... };` 부분만 복사합니다.

### 1-3. Firestore 데이터베이스 활성화 및 보안 세팅
1. 왼쪽 메뉴에서 **빌드(Build) > Firestore Database** 로 이동합니다.
2. **[데이터베이스 만들기]** 를 클릭하고 위치는 한국(Seoul - `asia-northeast3`)을 추천합니다.
3. `프로덕션 모드에서 시작`으로 설정하고 생성합니다.
4. 상단 탭 중 **[규칙(Rules)]** 탭으로 이동하여 기존 코드를 모두 지우고, 다음 보안 코드를 붙여넣고 **[게시]**를 클릭해 저장합니다.
*(이 규칙은 인프라.md 에서 정의한 비용 최소화 및 데이터 보호를 위한 기본 규칙입니다.)*

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // 커피라운지 조회 및 생성 권한 제한 (공통)
    match /coffeeLounge/{post} {
      // 읽기는 누구나 가능하나, 데이터 긁어가기 방지용으로 limit 강제 적용은 클라이언트 단에서 제어
      allow read: if true; 
      
      // 쓰기는 데이터 구조가 일치할 때만 허용
      allow create: if request.resource.data.keys().hasAll(['author', 'title', 'likes', 'comments', 'createdAt'])
                    && request.resource.data.title.size() > 0 
                    && request.resource.data.title.size() <= 500; 
      allow update: if false; // 작성된 글 수정 금지
      allow delete: if false; // 삭제 금지
    }
  }
}
```

### 1-4. 소스코드에 키 적용
에디터로 돌아와서 `산출물/js/dataService.js` 파일 상단의 `// TODO:` 아래에 복사해 둔 `firebaseConfig`를 덮어씁니다. (이후 로컬에서 브라우저를 띄워 피드 작성이 되는지 테스트합니다.)

## 2. Cloudflare Pages 배포 가이드 (웹 호스팅)

Firebase 연동이 끝났다면, Github에 올라간 최신 버전을 Cloudflare 로 배포해야 합니다. (Cloudflare Pages는 비용 효율적이며 무제한 트래픽에 매우 빠릅니다.)

### 2-1. Cloudflare Pages 연동
1. [Cloudflare 대시보드](https://dash.cloudflare.com/)에 로그인합니다.
2. 왼쪽 메뉴에서 **[Workers & Pages]** -> **[Overview]** 로 이동합니다.
3. 파란색 **[Create application]** 버튼 클릭 후, 상단 두 번째 탭인 **[Pages]** 를 선택합니다.
4. **[Connect to Git]** 을 클릭합니다.
5. GitHub 계정을 연동하고, 이 프로젝트 소스가 올라간 **저장소(Repository)** 를 선택합니다.

### 2-2. Build Settings 및 배포 설정
이 프로젝트는 복잡한 프레임워크 툴킷이 아니므로 별도 빌드 세팅이 필요 없습니다. 설정 화면에서 다음처럼 셋팅해주세요:

* **Framework preset (프레임워크 프리셋):** `None`
* **Build command (빌드 명령어):** *(비워둡니다)*
* **Build output directory (빌드 출력 디렉토리):** `산출물` 
  *(현재 소스코드가 전부 최상위가 아닌 `산출물` 폴더 내에 위치하기 때문에 이 경로를 써야 합니다)*

설정이 완료되면 **[Save and Deploy]** 버튼을 누릅니다.

### 2-3. (옵션) 사용자 정의 도메인 연결
1. 배포가 완료되면 `~.pages.dev` 형태의 임시 도메인이 주어집니다.
2. 우측 상단의 **[Custom domains]** 탭으로 이동하여 본인이 소유한 도메인(예: `aitomo.kr` 등)을 연동할 수 있습니다. (Cloudflare에 DNS가 등록되어 있어야 편합니다)
