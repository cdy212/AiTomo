# 데이터 처리 추상화 및 단계별 인프라 구축 플랜

현재 오픈을 위해 Firebase 기반으로 빠르게 연동하되, 차후 무료 VM DB(관계형 또는 기타 NoSQL)로 원활하게 전환하기 위한 아키텍처 및 개발 계획입니다.

## 1. 프론트엔드-데이터 계층 추상화 (1차 작업 준비 사항)
차후 DB를 교체하더라도 HTML/UI 요소는 단 한 줄도 변경하지 않기 위해 **Data Interface(추상화 레이어)** 를 구현합니다.

### `/js/dataService.js` 신규 생성
저장/조회 등을 실제로 수행하는 로직을 분리합니다. 1차적으로는 Firebase SDK를 이 파일 안에서만 사용하여 작동되게 세팅합니다.
모든 HTML 페이지에서는 `firebase` 함수를 직접 다루지 않고, 이 스크립트가 노출하는 아래의 범용 함수만 호출합니다.

- **주요 함수 형태:** 
  - `DataService.getPosts(collectionName, options)`: 게시글 목록 조회
  - `DataService.addPost(collectionName, requestData)`: 신규 글 등록
  - `DataService.updatePost(docId, requestData)`: 기존 글 수정
- **이점:** 나중에 이 파일 내의 함수 내용만 `fetch('VM서버 API 주소')`로 바꾸면 프론트엔드단 코드는 100% 그대로 동작합니다.

### `/js/firebaseInit.js` 신규 생성
Firebase 초기 설정 및 클라이언트 캐시(Offline Persistence 최적화 - '인프라.md' 규칙 준수용) 전용 로직을 담당합니다.

---

## 2. 인프라 세팅 (MCP Firebase 연동)
1. **Firebase Project 준비:** Firebase 프로젝트 생성 및 웹 앱 등록 후, 발급된 SDK Config 키값을 `firebaseInit.js` 에 주입.
2. **보안 규칙 (Security Rules) 설정:** 어뷰징을 막기 위해 Firestore 쓰기 제한 (본인 작성 글만 수정 가능, 데이터 크기 제한 등) 룰 반영.
3. **Storage 방어:** 필요 시 Cloudflare R2를 활용할 Presigned URL 생성 API 준비(Firebase Storage 사용 전면 차단 원칙 준수).

---

## 3. 차후 무료 VM & DB 전환 플랜 (2차 작업)

### 서버 구축 (Zero-Cost 백엔드)
- **추천 클라우드:** Oracle Cloud Free Tier (ARM 컴퓨팅 리소스 평생 무료) 또는 Serverless 컨테이너 제공 플랫폼(Render, Koyeb).
- **기술 스택:** 가벼운 Node.js Express 또는 Fastify, DB는 Neon(무료 PostgreSQL) 혹은 SQLite 이용.

### 코드 전환 프로세스 (무중단 마이그레이션)
1. 백엔드에서 `/api/posts` (GET, POST 등) 형태의 REST 구조 작성 완료 
2. 기존 프론트엔드의 `dataService.js` 코드를 열람
3. 함수 내부 로직을 Firebase 호출 형태에서 Browser 기본 `fetch API` 통신 형태로 교환
4. HTML 코드를 수정할 필요 없이 바로 백엔드를 거쳐 무료 DB로 쿼리가 전달되는 아키텍처 전환 완료
