# 동네꿀팁 (Neighborhood Tips) 기능 설계 플랜

> 작성일: 2026-04-22  
> 대상 파일: `dongnae.html` (신규 생성)  
> 연동 기존 모듈: `js/dataService.js`, `components/nav.html`, Firebase Firestore  

---

## 1. 개요 및 목표

**커피라운지 우측 사이드 패널** 혹은 **독립 페이지**로 "동네꿀팁" 섹션을 배치.  
사용자가 자신의 **현재 위치 기반**으로 주변의 육아 관련 장소(맛집·미용·놀이터·병원 등)를 지도로 확인하고, 직접 제보하며, 좋아요 기반 랭킹을 통해 신뢰도 높은 정보를 소비하는 커뮤니티 기반 로컬 정보 서비스.

---

## 2. 전체 아키텍처

```
[ 사용자 브라우저 ]
        |
        | Geolocation API (현재 위치 좌표 취득)
        |
[ dongnae.html ]
        |
        |---- 네이버 지도 SDK (지도 렌더링 / 마커 표시)
        |---- Naver Local Search API (상호명 검색) → CF Worker Proxy
        |---- Firebase Firestore (제보 데이터 저장/조회)
        |---- js/dataService.js (추상화 레이어 재사용)
```

### API 호출 경로 (보안)

```
브라우저 → Cloudflare Worker (CORS Proxy) → 네이버 Local Search API
```
네이버 API 키(Client-ID / Secret)는 절대 프론트에 노출하지 않고,
CF Worker 환경변수로만 관리한다. (기존 공통지침서 §5 준수)

## 5. Firestore DB 스키마

### Collection: `dongnaeTips`

```
dongnaeTips/
  └── {docId}
        ├── placeId        : string    // 네이버 장소 고유 ID
        ├── placeName      : string    // 상호명
        ├── category       : string    // 카테고리 코드
        ├── address        : string    // 도로명 주소
        ├── addressJibun   : string    // 지번 주소
        ├── lat            : number    // 위도
        ├── lng            : number    // 경도
        ├── naverMapUrl    : string    // 네이버 지도 링크 (선택)
        ├── memo           : string    // 메모 (선택)
        ├── reporterId     : string    // 제보자 uid 또는 'anonymous'
        ├── reporterNick   : string    // 닉네임
        ├── likes          : number    // 좋아요 수 (기본 0)
        ├── likedBy        : array     // 좋아요 uid 목록 (중복 방지)
        ├── images         : array     // 이미지 배열 (Base64)
        ├── createdAt      : timestamp
        └── status         : string    // 'active' | 'hidden'
```

## 6. 네이버 API 연동 상세

### 6-1. 사용 API

| API | 용도 | 비고 |
|-----|------|------|
| **네이버 Maps JavaScript API v3** | 지도 렌더링, 마커, 현재위치 | 클라이언트 직접 사용 (도메인 등록 필요) |
| **네이버 Local Search API** | 상호명으로 업체 검색 (최대 5건) | CF Worker Proxy 경유 필수 |

### 6-2. 네이버 지도 마커 렌더링 규칙

- 현재 위치 취득: `navigator.geolocation.getCurrentPosition()`
- 지도 초기 포커스: 현재 위치 좌표로 `map.setCenter(latlng)` 자동 이동
- 마커 아이콘: 카테고리별 색상 원형 SVG 인라인 마커 (가격 표시 X → 카테고리명 표시)
- 마커 클릭 시: 미니 InfoWindow 팝업 (장소명, 카테고리 뱃지, 좋아요 수)

### 6-3. Local Search API 프록시 응답 예시

```json
{
  "items": [
    {
      "title": "이룩빌라",
      "address": "서울특별시 동대문구 답십리동 956-34",
      "roadAddress": "서울특별시 동대문구 황물로15길 30-6",
      "mapx": "127123456",
      "mapy": "37123456",
      "link": "https://map.naver.com/..."
    }
  ]
}
```

## 7. 제보하기 화면 흐름

```
[+제보하기] 버튼 탭
        |
        ▼
Step 1: 상호명 검색
  - 검색창 입력 → CF Worker → Naver Local API
  - 결과 목록 (최대 5건) 표시
  - 선택 시 주소·좌표 자동 채워짐
        |
        ▼
Step 2: 상세 정보 입력
  - 카테고리 선택 (1개 이상 필수)
  - 네이버 지도 링크 (선택)
  - 메모 (선택)
  - 이미지 첨부 (최소 1장 필수)
        |
        ▼
Step 3: [등록] 버튼
  - Firestore 저장
  - 지도 마커 즉시 반영
```

**유효성 검사:**
- 업체는 검색 후 선택 필수 (직접 주소 입력 불가)
- 카테고리 1개 이상 선택
- 이미지 최소 1장 첨부

## 8. 랭킹 시스템

- **기준:** `likes` 필드 내림차순 정렬
- **범위:** 현재 위치 기준 반경 5km (클라이언트 거리 필터)
- **기본 상태:** 접힘(collapsed) → 클릭 시 아코디언 펼침
- **좋아요 중복 방지:** `likedBy` 배열에 uid 또는 로컬 fingerprint 저장
