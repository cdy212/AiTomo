# Supanova V3 프로젝트 공통 지침서 (가이드라인)

본 지침서는 육아 및 한일 커플/부부들의 커뮤니티 앱(V3) 개발 및 운영에 있어 준수해야 할 통합 룰과 CSS, UI/UX 가이드라인을 정의합니다. 모든 산출물 파일 수정 시 아래 룰을 준수해야 합니다.

## 0. 핵심 아이덴티티
- **브랜드명:** `AiTomo ' 아이친구 ' (아이토모 / あいとも)` 적용.
- **다국어 지원 의무:** 한국어와 일본어 사용자가 모두 접근하는 서비스이므로 네비게이션바 등 공통 영역에 언어 전환 제어기(Language Toggle)를 필수 배치합니다.

## 1. 최우선 UX 원칙: Voice-First & Hands-Free
- **손을 쓰기 힘든 타겟층 배려:** 아이를 돌보느라 양손이 바쁜 부모들이 주 타겟입니다. 타이핑(채팅)을 치기보다는 "익명질문"과 "정보 검색"에 필요한 음성 입력 버튼이 전역(어느 페이지든)에서 돋보여야 합니다.
- **다국어 음성 인식:** 마이크 입력 기능 연동 시, 반드시 사용자가 설정한 시스템 권장 언어(한국어인지 일본어인지)에 따라 음성을 판단해 처리하는 로직을 기본 전제로 둡니다.

## 2. 시각성 (Visuality & Typography)
- **이모지 제한 및 단색 아키텍처:** 기본 OS 이모지를 최대한 금지하고, 무채색 라인 아이콘인 `Iconify (Solar Icons)` 에 의존하여 통일감을 유지합니다.
- **히어로(메인) 이미지 조정:** 배경 이미지가 가려지지 않도록 박스 사이즈, `object-[position]` 및 투명도(opacity), 그라데이션 강도를 옅게 조정하여 중심 피사체(아기)가 직관적으로 잘리거나 한쪽으로 치우치지 않게 노출시킵니다.
- **텍스트 시안성 중심의 배열:** 모바일/데스크탑 환경에서 여백을 활용해 행 단위(List Matrix)로 데이터를 꽉 채우면서도 잘 떨어지는 비율을 유지합니다.

## 3. 웹 컴포넌트화 분리
- `/산출물/components/nav.html` 및 `voice_fab.html` 처럼 중복되는 코드는 JS의 `fetch()` API로 불러오며, 로컬 테스트 시 반드시 `run_server.bat` (파이썬 8000포트 기반) 등의 서버 스크립트를 통해 CORS 오류를 우회하여 테스트합니다. (file:/// 접근 금지)
- `cdn.tailwindcss.com`의 제작 환경 Warning은 프로토타이핑/시안성 검증 목적으로 무시하도록 약속합니다.

## 4. PWA(Progressive Web App) 접근성
- 타겟들이 휴대폰에서 손쉽게 기본 앱처럼 접근할 수 있도록 PWA 아키텍처를 도입합니다.
- `manifest.json`과 `sw.js`를 구성하여 모바일 홈 화면 바탕의 바로가기 추가가 가능하도록 하며 네이티브 앱 같은 심리스(Seamless)한 사용 환경을 유지합니다.
